# site-portal-pages

## 项目简介

site-portal-pages 是一个用于归档与发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态页面资源的统一存放与展示入口，方便在不同场景下快速引用或分发。

## 目录说明

```
site-portal-pages/
├── pages/          # 存放所有独立 HTML 页面文件
├── assets/         # 公共资源文件（如 CSS、JS、图片等）
├── index.html      # 页面索引或导航页（可选）
└── README.md       # 本文件
```

- **pages/**：每个子目录或文件对应一个独立的 HTML 页面，命名清晰，便于识别。
- **assets/**：存放页面共享的静态资源，避免重复引用。
- **index.html**：可选的入口页面，用于列出所有归档页面或提供简单导航。

## 页面归档说明

所有归档页面均为独立完整的 HTML 文件，可单独打开使用。页面之间无强依赖关系，资源引用均使用相对路径或内联方式，以确保离线或迁移时的可用性。

- 页面文件命名建议使用英文小写、连字符分隔，如 `example-page.html`。
- 每个页面应包含必要的元信息（如 `lang` 属性、`charset` 声明等）。
- 归档内容保持简洁，不包含跟踪代码、外部依赖或动态加载内容（除非必要）。

## 维护说明

- **新增页面**：在 `pages/` 目录下创建新的 HTML 文件，并更新 `index.html`（如果存在）。
- **更新页面**：直接修改对应文件，注意保持与其他页面或资源的兼容性。
- **删除页面**：移除文件及对应索引链接，确保无残留引用。
- **资源管理**：公共资源放入 `assets/`，按类型或用途分目录存放。

欢迎提交 Issue 或 Pull Request 以改进页面内容或结构。本仓库保持开源，遵循通用代码规范。
